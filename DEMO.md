# Demo Checklist

Use this checklist to publish a recruiter-friendly demo the next time hardware access is available.

## 90-Second Demo Structure

1. Show the robot platform and operator station.
2. Show the Qt GUI or operator interface receiving live camera data.
3. Send a short teleop sequence: forward, rotate, stop.
4. Pan and tilt the camera while the stream is live.
5. Show vitals values updating from the MAX30102 path.
6. End on a single slide or caption with the ROS 2 packages involved.

## Captioning Guidance

Keep the captions literal:
- command being sent
- topic or subsystem involved
- whether the segment is hardware-tested or software-only

That is more credible than a cinematic edit with no engineering context.
